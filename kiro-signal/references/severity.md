# SIGNAL Review Severity

| Level | Meaning |
|---|---|
| `5` | Breaks production, creates a security vulnerability, or risks data loss |
| `4` | Likely runtime error or crash under normal use |
| `3` | Wrong behavior under specific conditions |
| `2` | Maintainability issue, confusing pattern, or moderate code smell |
| `1` | Style, readability, or minor cleanup |

Assign severity by impact when triggered, not probability of triggering.
