# Claude skills: Code vs desktop / web (uploads)

SIGNAL is published as [Agent Skills](https://agentskills.io/)–compatible `SKILL.md` trees. **Where** you load them depends on which **Claude** product you use.

**Important distinction:** **Claude Code** usually uses **folders on disk** (personal, project, or plugin) — no ZIP. The **claude.ai / desktop “Customize → Skills → Upload”** flow often wants a **ZIP** whose archive contains a **`SKILL.md`** (and a folder name matching the `name:` in YAML, depending on the product version).

## Prebuilt ZIPs in this repo

Generated by [`scripts/pack-skill-zips-for-upload.ps1`](../scripts/pack-skill-zips-for-upload.ps1) into [`assets/`](../assets/):

| File | Contents | Use when |
| --- | --- | --- |
| `claude-skill-signal-flat.zip` | `SKILL.md` and `SKILL.min.md` at the **root** of the zip | The uploader rejects nested folders or is picky about path shape — try this first. |
| `claude-skill-signal-minimal.zip` | `signal/SKILL.md`, `SKILL.min.md` (forward slashes) | **Standard** `name` = folder = `signal` layout (matches Agent Skills). |
| `claude-skill-signal-with-references.zip` | `kiro-signal/skills/signal/…` and `kiro-signal/references/…` | You need **`../../references/...` links** in the mirrored `SKILL.md` to resolve. |

**“Zip file contains path with invalid characters”** — often caused by archives built with **Windows `Compress-Archive`**, which encodes path separators as **`\`**. The ZIP spec and many web uploaders only accept **`/`** in entry names. These assets are built with **Windows `tar -a` (zip)** so paths use `/`. If upload still fails, try **`claude-skill-signal-flat.zip`**, or repack yourself on macOS/Linux with `zip -r` from a clean tree.

**Regenerate after skill changes:** from repo root, run `sync-integration-packages.ps1`, then `pack-skill-zips-for-upload.ps1`.

## Claude Code “desktop” without a ZIP

On Windows, the **fast path** is still:

- **Plugin (if `/plugin` works):** see [README – Install, Claude Code](../README.md#install) and [`claude-signal/README.md`](../claude-signal/README.md).
- **No plugin / prefer folders:** run [`scripts/install-signal-all.ps1`](../scripts/install-signal-all.ps1) to copy skills into `%USERPROFILE%\.claude\skills\` (or copy `claude-signal/skills/signal` yourself).

**Zip uploads** are mainly for the **separate** Claude app **Skills** uploader, not a substitute for the Code CLI plugin system.
